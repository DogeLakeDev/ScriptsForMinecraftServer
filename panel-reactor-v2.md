panel/app.js:664-665 对监控、聊天、数据、模块页直接返回，早于全局 Esc 的 845-854。除数据表详情外，用户无法使用 Esc 返回总览，帮助文案与行为不一致。
5. Help/Confirm 不是完全模态  
✅
panel/app.js:594-612 只停止根 useInput；Ink 的子组件监听器仍活跃。MonitorView、ChatView、DbView、ModulesView 的 useInput 仍可在遮罩打开时改变滚动、焦点或搜索内容。应把 overlay 状态下传，并以 { isActive: false } 禁用局部输入。

panel/app.js:892-906 先向 BDS 发送 help，随后进入独立 if / else if 链的最终 else，再写入误导性警告。

P3 / 资源问题
 9. 监控 HTTP 请求超时后未显式销毁  
panel/monitor.js:177-197 配置了超时，但没有监听 timeout 并 destroy() 请求。db-server 卡死时可能保留挂起连接。建议改用统一 requestJson() 或显式销毁。
10. Chat 请求错误状态不完整  
panel/views/ChatView.js:75-89 消息请求异常被静默忽略；频道请求失败后 error 成功恢复时也未清空。用户可能看到旧消息，却无法判断刷新失败。
